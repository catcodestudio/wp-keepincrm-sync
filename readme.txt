Автоматична відправка замовлень WooCommerce у KeepinCRM: створення угоди в CRM після checkout, повторні спроби, журнал подій.

== Опис ==

Плагін зв'язує WooCommerce з KeepinCRM через публічне REST API (POST /agreements, заголовок X-Auth-Token). Кожне нове замовлення магазину автоматично створюється в KeepinCRM як угода з клієнтом-лідом, товарами (jobs) та коментарем.

= Можливості =

* Відправка замовлення одразу після checkout (класичний чекаут і Checkout Blocks) або при переході в один із вибраних статусів
* Дедуплікація: збережений KeepinCRM ID угоди у меті замовлення — жодних дублів
* Клієнт створюється як лід (client_attributes: ПІБ, email, телефон), товари передаються рядками угоди (jobs_attributes)
* Необов'язкова маршрутизація: ID воронки, етапу, джерела та відповідального працівника
* Позначка оплати: для сплачених онлайн замовлень у коментар угоди додається «✅ Оплачено онлайн» (опційно)
* Повторні спроби при збої: 3 спроби з паузами 5 хв / 30 хв / 2 год (WP-Cron)
* Метабокс на сторінці замовлення: статус синхронізації, KeepinCRM ID, кнопка «Відправити повторно» (для вже відправлених — оновлення через PATCH /agreements/{id})
* Кнопка «Перевірити з'єднання» на сторінці налаштувань (GET /clients/statuses)
* Журнал останніх подій синхронізації (вкладка «Журнал»)
* Шифрування API-токена у БД (libsodium з HMAC-фолбеком)
* Нормалізація телефону покупця до E.164 (+380...)
* HPOS-сумісність (custom order tables), робота лише через WC_Order CRUD
* Фільтр `cc_keepincrm_order_payload` і хук `cc_keepincrm_order_sent` для сторонніх інтеграцій

= Вимоги =

* WooCommerce 7.0 або новіше
* PHP 7.4+
* API-токен KeepinCRM (кабінет → Налаштування → Інтеграції → API)

= Як це працює =

1. Клієнт оформлює замовлення у магазині
2. Плагін збирає угоду (клієнт, товари-рядки, сума, коментар) і надсилає POST /agreements у KeepinCRM (base https://api.keepincrm.com) із заголовком X-Auth-Token
3. KeepinCRM повертає 201 з `{"id":N}` — ID угоди зберігається у меті `_cc_keepincrm_agreement_id`
4. Якщо KeepinCRM недоступний — спроба повторюється автоматично (до 3 разів)
5. Менеджер бачить статус і ID у метабоксі замовлення та в журналі

== Встановлення ==

1. Залити папку плагіна у `/wp-content/plugins/keepincrm-sync-for-woocommerce/`
2. Активувати у меню «Плагіни»
3. WooCommerce → KeepinCRM Sync → ввести API-токен
4. Натиснути «Перевірити з'єднання», зберегти налаштування

== Часті питання ==

= Звідки взяти API-токен? =

Кабінет KeepinCRM → Налаштування → Інтеграції → API. Токен передається у заголовку X-Auth-Token і зберігається у БД в зашифрованому вигляді.

= Чи створюються дублі при повторній зміні статусу? =

Ні. Плагін зберігає KeepinCRM ID угоди у меті замовлення і перед створенням перевіряє його — повторна відправка того самого замовлення оновлює наявну угоду, а не створює нову.

= Що робить кнопка «Відправити повторно»? =

Якщо замовлення ще не в KeepinCRM — створює угоду (POST /agreements). Якщо вже відправлене — оновлює її через PATCH /agreements/{id} актуальними даними.
